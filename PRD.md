# PRD（极简MVP）- OnTheWay（到了吗？）

## 1. 目标
做一个单网页工具，输入多人出发地后，给出一个建议碰面点，并在地图上展示。

## 2. MVP范围（只做这4件事）
1. 单页 `index.html`（不做登录、不做数据库）。
2. 输入：3-8人出发地、聚会时间、可选口味偏好。
3. 分析接口：调用 `DeepSeek API`，返回1个推荐碰面点（名称+理由）。
4. 地图接口：调用 `高德地图 API`，展示推荐点和各出发点标记（可选显示路线/耗时）。

## 3. 非范围（本期不做）
1. 实时定位与“谁已出发”状态同步。
2. 多候选地点排序与复杂权重策略。
3. 餐厅预订、消息通知、账号体系。

## 4. 用户流程
1. 输入聚会时间、成员出发地、口味偏好。
2. 点击“生成碰面点”。
3. 页面展示：推荐地点 + 推荐理由 + 地图标记。

## 5. 最小功能要求
1. 至少支持3个出发地输入。
2. DeepSeek返回可读的推荐地点文本。
3. 高德地图可正常加载并显示标记点。
4. 失败场景有提示（如API失败、地址无效）。

## 6. 技术约束
1. 前端：原生 `HTML + JavaScript`。
2. AI Key：`DEEPSEEK_API_KEY`。
3. 地图 Key：`AMAP_API_KEY`。
4. 本地演示优先，先跑通再优化安全性。

## 7. 验收标准（Done）
1. 从输入到结果可完整跑通。
2. 能稳定输出1个推荐碰面点。
3. 地图上能看到推荐点与出发点。
4. 全流程在正常网络下10秒内出结果（目标值）。

## 8. 1小时执行计划
1. 0-15分钟：页面表单。
2. 15-30分钟：接 DeepSeek 推荐。
3. 30-50分钟：接高德地图标记。
4. 50-60分钟：联调与演示检查。
