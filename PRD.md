# PRD（1小时MVP版）- OnTheWay（到了吗？）

## 1. 产品目标
在一个网页里，让多人输入出发地，自动给出一个建议碰面点，并在 Google Map 上展示。

## 2. 目标用户
3-8 人的聚会组织者（约饭、见面）。

## 3. MVP范围
1. 单页网页 `index.html`。
2. 输入：聚会时间、每个人出发地、可选口味偏好。
3. 调用 Gemini API：返回 1 个推荐碰面地点（名称+简短理由）。
4. Google Maps 展示：
- 推荐地点标记
- 出发地点标记
- 简单路线/预计耗时（能展示即可）

## 4. 非范围（本次不做）
1. 登录与账号体系。
2. 实时位置共享与“已出发”状态同步。
3. 多候选地点排序与复杂算法。
4. 后端数据库持久化。

## 5. 核心用户流程
1. 用户打开网页。
2. 输入 3-8 个出发地 + 聚会时间 + 口味偏好。
3. 点击“生成碰面点”。
4. 页面展示推荐地点与地图标注。
5. 用户据此决定集合地点。

## 6. 功能需求
1. 表单模块：
- 可增减成员输入行（最少3人）。
- 基础校验：地址不能为空。

2. AI推荐模块：
- 请求参数：出发地列表、时间、偏好。
- 返回：推荐地点名称、地址、推荐理由（1-2句）。

3. 地图模块：
- 地图初始化成功。
- 显示推荐点与成员出发点 marker。
- 显示路线或预计耗时（至少一条成功即可）。

4. 异常处理：
- API失败时提示“请重试”。
- 地址解析失败时提示“请检查地址格式”。

## 7. 技术方案
1. 前端：原生 `HTML + CSS + JavaScript`。
2. AI：`Gemini API`。
3. 地图：`Google Maps JavaScript API`。
4. 运行方式：本地直接打开或本地静态服务运行。

## 8. 验收标准（Done）
1. 能输入至少3个出发地并提交。
2. 能返回1个推荐碰面点。
3. 地图能成功显示并标记地点。
4. 正常网络下，10秒内看到结果。

## 9. 1小时执行计划
1. 0-10分钟：页面与表单。
2. 10-25分钟：接 Gemini 返回推荐点。
3. 25-45分钟：接 Google Maps 并打点。
4. 45-60分钟：联调、报错提示、演示走查。
